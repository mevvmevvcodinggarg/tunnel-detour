# Security Policy

## Supported version

Security fixes target the latest TunnelDetour Beta.

## Reporting a vulnerability

Do not open a public issue for a suspected privilege-escalation, command-execution,
or sensitive-data vulnerability. Use the repository's
[private vulnerability reporting](https://github.com/mevvmevvcodinggarg/tunnel-detour/security/advisories/new)
feature and include a minimal reproduction, impact, and affected version.

Remove private domains, VPN names, addresses, credentials, employer/customer
identifiers, and unrelated logs. Maintainers will acknowledge a valid private
report through GitHub and coordinate a fix before public disclosure.
